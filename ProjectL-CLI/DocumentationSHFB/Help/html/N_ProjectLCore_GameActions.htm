      ProjectLCore.GameActions Namespace

       

      Classes

      ActionVerifier Verifies the validity of actions in the context of the current game state.
      CannotPlaceTetrominoFail The specified tetromino cannot be placed into the puzzle with the given ID.
      DoNothingAction Last resort action for AI players, they should never actually need to use it. It will always be accepted unless the game phase is [!:GamePhase.FinishingTouches].
      EmptyRowRecycleFail The player tried to recycle an empty row.
      EndFinishingTouchesAction Represents the action of ending a player's turn during [!:GamePhase.FinishingTouches]
      GameActionProcessor A class for processing player actions in the game. One instance should be created for each player. The class is responsible for updating the game state based on the player's actions. It isn't responsible for verifying the actions. The actions should be verified by ActionVerifier before being processed.
      ChangeTetrominoAction Represents the action of changing a tetromino for a different one.
      InvalidActionDuringFinishingTouchesFail The player used an invalid action during FinishingTouches. The only allowed actions are PlaceTetrominoAction and EndFinishingTouchesAction.
      InvalidEndFinishingTouchesActionUseFail The player used the EndFinishingTouchesAction during a different phase than FinishingTouches.
      InvalidTetrominoConfigurationFail The given configuration doesn't match the given shape.
      InvalidTetrominoChangeFail It is not possible to change the old tetromino for the new one.
      MasterAction Represents the use of the Master Action.
      MasterActionAlreadyUsedFail The player has already used the MasterAction in this turn.
      MasterActionNotEnoughTetrominosFail The player doesn't have enough tetrominos needed by a MasterAction.
      MasterActionUniquePlacementFail Two of the placements specified by a Master action are to the same puzzle.
      NumberOfRecycledPuzzlesMismatchFail The player tried to recycle the puzzles of the given color, but the number of puzzles to recycle doesn't match the number of puzzles in the row.
      PlaceTetrominoAction Represents the action of placing a tetromino on a puzzle.
      PlayerAlreadyTookBlackPuzzleInEndOfTheGameFail The player tried to take a second black puzzle in the same turn during EndOfTheGame.
      PlayerDoesntHavePuzzleFail The player doesn't have the puzzle with this ID.
      PuzzleDeckIsEmptyFail The player specified TopBlack or TopWhite but the specified deck is empty.
      PuzzleIdIsNullFail The player specified Normal but PuzzleId was null.
      PuzzleNotAvailableFail The player tried to take a puzzle which isn't available.
      PuzzleNotInRowFail There is an ID in Order which doesn't match any puzzle in specified row.
      RecycleAction Represents the action of recycling puzzles. The player chooses a row to recycle. The puzzles from the row will be put to the bottom of the deck in the order specified by the player. The puzzle row is then refilled.
      TakeBasicTetrominoAction Represents the action of taking a O1 tetromino from the shared reserve.
      TakePuzzleAction Represents the action of taking a puzzle. Players can take puzzles from the top of the white deck, top of the black deck or a specific puzzle in one of the rows.
      TetrominoAction The base class for TakeBasicTetrominoAction and ChangeTetrominoAction because they are technically the same action, just with different parameters.
      TetrominoNotInPersonalSupplyFail The player doesn't have any tetrominos of the given shape.
      TetrominoNotInSharedReserveFail There are no tetrominos of this shape left in the shared reserve.
      VerifiableAction Represents an action which can be verified. All subclasses which inherit from this class should be immutable. This ensures that the action can not be changed after it has been created and therefore it's Status can be trusted.
      VerificationFailure Represents a failed verification and provides a message describing the failure. Derived classes should provide more context.
      VerificationStatus Represents the result of the verification.
      VerificationSuccess Represents a successful verification.
